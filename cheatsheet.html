Query Execution Order

This is the order SQL thinks in — not the order you write it:

StepClauseWhat it does
1FROM / JOINPick tables, combine rows
2WHEREFilter individual rows
3GROUP BYCollapse rows into groups
4HAVINGFilter groups
5SELECTChoose columns, compute expressions
6ORDER BYSort results
This is why you can't use aliases from SELECT in WHERE — SELECT runs after WHERE.

JOIN Types

TypeWhat it keepsUse when
INNER JOINOnly matching rows from both"Show customers who have rentals"
LEFT JOINAll left + matched right (NULL if no match)"Show ALL customers, even with 0 rentals"
CROSS JOINEvery combination (m × n)Generating all possible pairs
SELF JOINSame table twice with different aliases"Find pairs sharing same attribute"
LEFT JOIN trap: After LEFT JOIN, use COUNT(right_table.id) not COUNT(*). COUNT(*) counts the NULL-padded row as 1.

CASE Expression

CASE
  WHEN amount > 10 THEN 'High'    -- most restrictive FIRST
  WHEN amount > 5  THEN 'Medium'
  WHEN amount > 0  THEN 'Low'
  ELSE 'Free'                     -- always include ELSE
END AS category
Order matters! CASE evaluates top-to-bottom and stops at the first TRUE. Put the most restrictive condition first.

Conditional Aggregation (Pivot Pattern)

-- Count how many large payments
SUM(CASE WHEN amount > 10 THEN 1 ELSE 0 END) AS big_count

-- Sum the value of large payments
SUM(CASE WHEN amount > 10 THEN amount ELSE 0 END) AS big_total

-- Pivot rows to columns
SUM(CASE WHEN month = 'Jan' THEN revenue ELSE 0 END) AS jan_rev,
SUM(CASE WHEN month = 'Feb' THEN revenue ELSE 0 END) AS feb_rev

Subquery Shapes

ShapeReturnsUse with
Scalar1 row × 1 col=, <, >, BETWEEN
Listmany rows × 1 colIN, NOT IN, EXISTS
Tablemany rows × many colsFROM (...) t (derived table)
NOT IN + NULL = empty result! If the subquery can return NULLs, use NOT EXISTS instead or add WHERE col IS NOT NULL inside the subquery.

EXISTS vs IN

-- EXISTS: "does at least one matching row exist?"
WHERE EXISTS (SELECT 1 FROM rental r WHERE r.customer_id = c.customer_id)

-- IN: "is this value in the list?"
WHERE customer_id IN (SELECT customer_id FROM rental)

-- Use EXISTS when NULLs are a concern — it's NULL-safe

Self JOIN Pattern

-- Find pairs (e.g., actors with same last name)
SELECT a1.first_name, a1.last_name, a2.first_name, a2.last_name
FROM actor a1
JOIN actor a2 ON a1.last_name = a2.last_name
WHERE a1.actor_id < a2.actor_id;   -- use < not <>
Use < not <> — this prevents both self-matches AND duplicate pairs in one condition.

Derived Tables

-- "Customers who spent more than average customer"
SELECT * FROM (
  SELECT customer_id, SUM(amount) AS total
  FROM payment GROUP BY customer_id
) t
WHERE total > (
  SELECT AVG(total) FROM (
    SELECT customer_id, SUM(amount) AS total
    FROM payment GROUP BY customer_id
  ) x
);
Every derived table needs an alias: (...) t. Without it, you get an error.
